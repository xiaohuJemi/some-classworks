# Agent 能做什么（仓库内说明）

面向首次使用者的简要说明，涵盖能力、权限与基础配置需求。

## 能力范围
- 阅读与搜索仓库文件，解释结构与代码含义
- 新增、修改、删除文件并提交变更
- 运行项目中已有的命令（如测试、构建、脚本）
- 编写说明文档、改进代码或排查问题
- 依据需要使用 GitHub 工作流/PR 信息进行分析

## 需要的权限或基础配置
- **仓库读写权限**：用于读取与保存文件
- **命令执行权限**：允许运行项目已有的构建/测试/脚本命令
- **依赖安装能力**：可通过包管理器安装项目依赖（按项目需要）
- **GitHub 认证能力**：用于提交变更与创建 PR（若需要）
- **网络访问**：下载依赖或获取外部资料时可能需要（部分外网可能受限）

## 运行限制（透明说明）
- 不能直接 `git push`，需通过受控工具提交与推送
- 不会访问其他仓库或本机隐私文件
- 不会写入或泄露敏感信息（密钥、令牌等）
- 外部网络访问可能受限，必要时需提供允许访问的地址

## 你需要提供的信息
- 目标与期望产出（文件名、位置、格式）
- 相关代码/文件路径或业务背景
- 构建/测试命令（若仓库未提供说明）
- 任何环境依赖或版本约束
